# No.387

## 問題：デプロイ順序の強制（最小の追加依存）

サービスは `0..n-1` の `n` 個です。

現在の依存関係が `edges` として与えられます。`edges[i] = [a, b]` は **a を先にデプロイしてから b をデプロイする必要がある**（有向辺 `a -> b`）ことを意味します。

あなたは希望するデプロイ順序 `order`（長さ n、0..n-1 の順列）を持っています。

あなたは依存関係に「追加の辺」をいくつでも追加できます（既存の辺は削除できません）。

**目的：追加後の依存関係において、デプロイ可能な順序が必ず `order` の順番になるように “強制” したい**です。
（言い換えると、任意のデプロイ手順は `order` と矛盾しない必要がある）

この目的を満たすために必要な **追加辺の最小本数**を求めてください。

もし既存の依存関係が原因で `order` を強制できない（`order` と矛盾する辺がある）場合は `-1` を返してください。

* `1 <= n <= 2 * 10^5`
* `0 <= len(edges) <= 2 * 10^5`
* `order` は 0..n-1 の順列

**求めるもの**：追加すべき辺の最小本数（整数）、不可能なら `-1`

## 実行例

例1:

* 入力:

  * `n = 4`
  * `edges = [[0,2], [1,2]]`
  * `order = [0,1,2,3]`
* 説明:

  * `order` を強制するには、最低でも隣接関係 `0->1`, `1->2`, `2->3` が必要
  * 既に `1->2` はあるので、追加は `0->1` と `2->3` の **2本**
* 出力: `2`

例2:

* 入力:

  * `n = 3`
  * `edges = [[2,0]]`
  * `order = [0,1,2]`
* 説明:

  * 既存の辺 `2->0` が `order` と矛盾（2は0より後なのに先に必要）
  * 辺は削除できないので不可能
* 出力: `-1`

例3:

* 入力:

  * `n = 4`
  * `edges = [[0,2], [2,3]]`
  * `order = [0,2,1,3]`
* 説明:

  * 隣接関係として必要なのは `0->2`, `2->1`, `1->3`
  * `0->2` は既にあるので、追加は `2->1` と `1->3` の **2本**
* 出力: `2`

## 実行結果

```
2
-1
2
```
