## 問題: 連続するログ区間の「最小差」を求めよ（Minimum Absolute Difference in a Sliding Window）

SRE のメトリクス監視を想定します。

整数配列 `nums` と整数 `k` が与えられます。

任意の連続部分配列（ウィンドウ）長さ `k` について、そのウィンドウ内の **任意の2要素の差の絶対値**の最小値を

* `minDiff(window) = min_{i≠j} |window[i] - window[j]|`

と定義します。

> すべての長さ `k` のウィンドウのうち、`minDiff(window)` の **最大値**を返してください。

つまり「どの区間を見ても、値が互いに離れていて（近い値が存在しない）、その“近さ”の最悪ケースを最大化する」問題です。

### 制約（目安）

* `1 ≤ k ≤ len(nums) ≤ 10^5`
* `-10^9 ≤ nums[i] ≤ 10^9`

時間計算量は **O(n log k)** を目指してください。

## 実行例

```python
# 例1
nums = [1, 5, 9, 2, 6]
k = 3
# ウィンドウ:
# [1,5,9]   -> 最小差 min(|1-5|,|5-9|,|1-9|)=4
# [5,9,2]   -> ソート [2,5,9]  最小差=3
# [9,2,6]   -> ソート [2,6,9]  最小差=3
# これらの minDiff の最大は 4
# 出力: 4

# 例2
nums = [10, 11, 12, 13]
k = 2
# 各ウィンドウの最小差は [10,11]=1, [11,12]=1, [12,13]=1
# 最大は 1
# 出力: 1

# 例3
nums = [3, 3, 3, 3]
k = 3
# どのウィンドウも最小差 0
# 出力: 0
```

## 実行結果

```

例1 出力: 4 (期待値: 4)
------------------------------------------------------------
例2 入力: nums=[10, 11, 12, 13], k=2
例2 出力: 1 (期待値: 1)
------------------------------------------------------------
例3 入力: nums=[3, 3, 3, 3], k=3
例3 出力: 0 (期待値: 0)
------------------------------------------------------------
```
