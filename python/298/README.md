## 問題: 合計が目標値以上となる最短部分配列の長さ

正の整数からなる配列 `nums` と、正の整数 `target` が与えられます。

> 「**合計が `target` 以上になる連続部分配列**」のうち、
> **長さが最も短いものの長さ** を求めてください。

* 部分配列とは、配列の連続した区間のことです。
* 該当する部分配列が存在しない場合は `0` を返してください。

### 制約（目安）

* `1 ≤ len(nums) ≤ 10^5`
* `1 ≤ nums[i] ≤ 10^5`
* `1 ≤ target ≤ 10^9`

`nums` が **すべて正の整数** であることを利用して、
時間計算量 **O(n)** の解法（スライディングウィンドウ）を意識してください。

## 実行例

```python
# 例1
target = 7
nums = [2, 3, 1, 2, 4, 3]
# 合計が7以上になる連続部分配列は:
# [2,3,1,2] (長さ4, 和8)
# [3,1,2,4] (長さ4, 和10)
# [1,2,4]   (長さ3, 和7)
# [4,3]     (長さ2, 和7) ← 最短
# 出力: 2

# 例2
target = 4
nums = [1, 4, 4]
# 合計が4以上になる最短部分配列は [4] (インデックス1 or 2)
# 長さは 1
# 出力: 1

# 例3
target = 11
nums = [1, 2, 3, 4, 5]
# 合計が11以上になる部分配列:
# [3,4,5] (長さ3, 和12)
# [4,5]   (長さ2, 和9) は 11 未満なのでNG
# 最短は [3,4,5] で長さ3
# 出力: 3

# 例4
target = 100
nums = [1, 2, 3]
# どの部分配列も和が100以上にならない
# 出力: 0
```

## 実行結果

```
例1の入力: target = 7 nums = [2, 3, 1, 2, 4, 3]
例1の結果: 2
例2の入力: target = 4 nums = [1, 4, 4]
例2の結果: 1
例3の入力: target = 11 nums = [1, 2, 3, 4, 5]
例3の結果: 3
例4の入力: target = 100 nums = [1, 2, 3]
例4の結果: 0
```
