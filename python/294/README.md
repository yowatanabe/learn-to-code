# No.294

## 問題: 回転されたソート配列の検索

**昇順にソートされた整数配列** `nums` が、ある位置で **1 回だけ回転** されています。
（例: 元の配列が `[0,1,2,4,5,6,7]` のとき、回転後は `[4,5,6,7,0,1,2]` など）

* 配列 `nums` の要素は **すべて異なる** とします。
* 整数 `target` が与えられたとき、`target` が配列 `nums` の中に存在するなら **そのインデックス** を返してください。
* 存在しない場合は `-1` を返してください。

アルゴリズムの時間計算量は、**O(log n)** を目指してください（線形探索 O(n) は不採用想定）。

### 制約（目安）

* `1 ≤ len(nums) ≤ 10^4`
* `-10^4 ≤ nums[i] ≤ 10^4`
* `nums` の要素はすべて異なる
* `-10^4 ≤ target ≤ 10^4`

## 実行例

```python
# 例1
nums = [4,5,6,7,0,1,2]
target = 0
# 0 はインデックス 4 に存在する
# 出力: 4

# 例2
nums = [4,5,6,7,0,1,2]
target = 3
# 3 は存在しない
# 出力: -1

# 例3
nums = [1]
target = 0
# 配列に 0 は存在しない
# 出力: -1

# 例4
nums = [1]
target = 1
# 1 はインデックス 0
# 出力: 0
```

## 実行結果

```
例1の入力: [4, 5, 6, 7, 0, 1, 2] target = 0
例1の結果: 4
例2の入力: [4, 5, 6, 7, 0, 1, 2] target = 3
例2の結果: -1
例3の入力: [1] target = 0
例3の結果: -1
例4の入力: [1] target = 1
例4の結果: 0
```
