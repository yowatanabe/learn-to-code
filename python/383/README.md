# No.383

## 問題文（LeetCode風・日本語）

**問題：ログIDの重複除去（長さ k の連続重複禁止）**

監視ログのイベントID列 `ids` が与えられます。あなたは次のルールでログを間引いて「送信ログ」を作ります。

左から順に `ids[i]` を見て、送信ログ `out` に追加するかを決めます。

* `out` の直近 `k` 件（末尾側、件数が `k` 未満なら全件）の中に **同じIDが1つでも存在する** 場合、`ids[i]` は **捨てる**
* そうでなければ `out` に **追加する**

最終的な `out` を返してください。

* `1 <= len(ids) <= 2 * 10^5`
* `1 <= k <= 2 * 10^5`
* `ids[i]` は整数（負でもよい）
* 時間計算量は **O(n)** を目指してください

**求めるもの**：フィルタ後の配列 `out`

## 実行例

例1:

* 入力: `ids = [1, 2, 1, 3, 1, 2]`, `k = 2`
* 処理:

  * out=[] → 1追加 → [1]
  * 2追加 → [1,2]
  * 次の1は直近2件[1,2]に1がいる → 捨てる
  * 3は直近2件[1,2]にいない → 追加 → [1,2,3]
  * 次の1は直近2件[2,3]にいない → 追加 → [1,2,3,1]
  * 次の2は直近2件[3,1]にいない → 追加 → [1,2,3,1,2]
* 出力: `[1, 2, 3, 1, 2]`

例2:

* 入力: `ids = [5, 5, 5, 5]`, `k = 1`
* 説明: 直近1件に同じIDがあれば捨てるので、連続重複が消える
* 出力: `[5]`

例3:

* 入力: `ids = [1, 2, 3, 1, 2, 3]`, `k = 3`
* 説明: 直近3件に同じIDがあると捨てるため、後半の1,2,3はすべて捨てられる
* 出力: `[1, 2, 3]`

## 実行結果

```
[1, 2, 3, 1, 2]
[5]
[1, 2, 3]
```
