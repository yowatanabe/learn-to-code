# No.394

## 問題：通信経路の最短ホップ数

あなたは SRE として、サービス間通信の経路を調査しています。

`n` 個のサービスがあり、`0` から `n-1` までの番号が付いています。

サービス間の接続情報が `edges` として与えられます。

`edges[i] = [u, v]` は、サービス `u` とサービス `v` の間を**双方向**に通信できることを表します。

開始サービス `start` から終了サービス `target` まで到達するために必要な**最小ホップ数**を求めてください。

到達できない場合は `-1` を返してください。

* 1回の通信で、隣接する1つのサービスへ移動できる
* `1 <= n <= 2 * 10^5`
* `0 <= len(edges) <= 2 * 10^5`
* `0 <= start, target < n`
* 同じ辺が複数回与えられることはありません

**求めるもの**：`start` から `target` までの最小ホップ数（整数）

## 実行例

例1:

* 入力:

  * `n = 6`
  * `edges = [[0,1],[0,2],[1,3],[2,3],[3,4],[4,5]]`
  * `start = 0`
  * `target = 5`
* 説明:

  * 例えば `0 -> 1 -> 3 -> 4 -> 5` で到達できる
  * ホップ数は `4`
* 出力:

  * `4`

例2:

* 入力:

  * `n = 4`
  * `edges = [[0,1],[2,3]]`
  * `start = 0`
  * `target = 3`
* 説明:

  * 0 から 3 へ到達できない
* 出力:

  * `-1`

例3:

* 入力:

  * `n = 5`
  * `edges = [[0,1],[1,2],[2,3],[3,4]]`
  * `start = 2`
  * `target = 2`
* 説明:

  * 開始地点と終了地点が同じなのでホップ数は `0`
* 出力:

  * `0`

## 実行結果

```
4
-1
0
```
