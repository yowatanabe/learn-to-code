## 問題: K個に分割して最大値を最小化せよ（Minimize the Maximum After Splitting）

非負整数配列 `nums` と整数 `k` が与えられます。あなたは **高々 `k` 回**、次の操作を行えます：

* `nums[i]` を選び、正の整数 `a, b` に分割して `a + b = nums[i]` を満たすように置き換える
  （例: `10` を `6` と `4` に分割する）

操作後の配列における **最大値** をできるだけ小さくしたいです。

> 操作を高々 `k` 回行った後に達成できる「配列の最大値」の最小値を返してください。

### 補足

* 1回の操作で要素数は 1 つ増えます（1つを2つにするため）
* 分割は何度でも同じ要素に対して行えます（結果として 10 → 4+3+3 のようにもできる）

### 制約（目安）

* `1 ≤ len(nums) ≤ 10^5`
* `0 ≤ nums[i] ≤ 10^9`
* `0 ≤ k ≤ 10^9`

期待解法: **二分探索 + 貪欲チェック**（`O(n log(max(nums)))`）

## 実行例

```python
# 例1
nums = [9]
k = 2
# 9 を 3+3+3 にするには 2回分割（9->6+3, 6->3+3）
# 最大値は 3 が最小
# 出力: 3

# 例2
nums = [7, 2, 5]
k = 1
# 7 を 4+3 に分割すると最大は max(4,3,2,5)=5
# これが最小
# 出力: 5

# 例3
nums = [1, 1, 1]
k = 10
# すでに最大1。これ以上下げられない
# 出力: 1
```

------------------------------------------------------------
