# No.378

## 問題文（LeetCode風・日本語）

**問題：障害インシデントのまとめ上げ（最小マージ回数）**

ある監視システムでは、インシデントが発生すると時系列にイベントが記録されます。

ただしノイズが多く、**近い時間に発生したイベントは同一インシデントとしてまとめたい**です。

昇順にソートされた整数配列 `t`（イベント発生時刻、単位：秒）と整数 `k` が与えられます。

次の操作を何度でも行えます：

* 隣り合う2つのグループ（連続区間）を1つにマージする。マージ後のグループの時刻範囲は、両グループの最小時刻〜最大時刻を含む。

最終的に、すべてのグループが **「最大時刻 - 最小時刻 <= k」** を満たすようにしたいです。

このとき必要な**最小マージ回数**を求めてください。

* 初期状態では、各イベントはそれぞれ1つのグループ（つまり `len(t)` 個のグループ）
* `1 <= len(t) <= 2 * 10^5`
* `0 <= k <= 10^9`
* `t` は昇順（同値あり）

**求めるもの**：最小マージ回数（整数）

## 実行例

例1:

* 入力: `t = [1, 2, 4, 10, 11]`, `k = 3`
* 説明:

  * `[1,2,4]` は幅 `4-1=3` でOK
  * `[10,11]` は幅 `1` でOK
  * 初期は5グループ → 最終は2グループなのでマージ回数は `5-2=3`
* 出力: `3`

例2:

* 入力: `t = [1, 5, 9]`, `k = 2`
* 説明:

  * どの2つをマージしても幅が2を超えるので、最終的に全てを満たすことは不可能…ではなく、
    「各グループが幅<=k」を満たすようにするためには **マージしない** のが最良（各1点の幅は0）
  * 初期から条件を満たしているのでマージ回数0
* 出力: `0`

例3:

* 入力: `t = [1, 2, 100, 101, 102]`, `k = 1`
* 説明:

  * `[1,2]` は幅1でOK
  * `[100,101]` は幅1でOK
  * `[102]` は幅0でOK
  * 最終は3グループなのでマージ回数は `5-3=2`
* 出力: `2`

## 実行結果

```
3
0
2
```
