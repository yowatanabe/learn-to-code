# No.402

## 問題：最小ヒープ使用量でのジョブ実行

あなたは SRE として、複数のバッチジョブを実行します。

各ジョブは **開始時刻** と **終了時刻** を持ち、実行中は 1 台のワーカーを専有します。

ジョブの配列 `jobs` が与えられます。

`jobs[i] = [start_i, end_i]` は、ジョブ `i` が時刻 `start_i` から時刻 `end_i` **未満** まで実行されることを表します。

同じ時刻に複数のジョブが重なっている場合、それぞれ別のワーカーが必要です。

すべてのジョブを実行するために必要な **最小ワーカー数** を求めてください。

* `1 <= len(jobs) <= 2 * 10^5`
* `0 <= start_i < end_i <= 10^9`

**求めるもの**：必要な最小ワーカー数（整数）

## 実行例

例1:

* 入力: `jobs = [[0, 30], [5, 10], [15, 20]]`
* 説明:

  * `[0,30)` の実行中に `[5,10)` が重なるので 2 台必要
  * `[15,20)` の開始時点では `[5,10)` は終了しているため、同じワーカーを再利用できる
* 出力: `2`

例2:

* 入力: `jobs = [[7, 10], [2, 4]]`
* 説明:

  * ジョブは重ならないので 1 台で足りる
* 出力: `1`

例3:

* 入力: `jobs = [[1, 5], [2, 6], [4, 8], [9, 10]]`
* 説明:

  * 時刻 4 以上 5 未満の間は 3 ジョブが同時実行される
  * よって 3 台必要
* 出力: `3`

## 実行結果

```
2
1
3
```
