# No.293

## 問題: 和が `k` となる連続部分配列の個数

整数配列 `nums` と整数 `k` が与えられます。

> 「**合計がちょうど `k` になる連続部分配列**」の個数を求めてください。

* 部分配列とは、配列の連続した区間のことです。
* 空の部分配列は数えません。
* 答えは 32bit 整数に収まると仮定してよいものとします。

### 制約（目安）

* `1 ≤ len(nums) ≤ 2 * 10^4`
* `-10^3 ≤ nums[i] ≤ 10^3`
* `-10^7 ≤ k ≤ 10^7`

時間計算量 **O(n)**、追加メモリ **O(n)** 程度の解法を目指してください
（全探索 O(n^2) は TLE 想定）。

## 実行例

```python
# 例1
nums = [1, 1, 1]
k = 2
# 和が 2 になる連続部分配列:
# [1, 1] (インデックス 0〜1)
# [1, 1] (インデックス 1〜2)
# 出力: 2

# 例2
nums = [1, 2, 3]
k = 3
# 和が 3 になる連続部分配列:
# [1, 2] (1+2=3)
# [3]    (3)
# 出力: 2

# 例3
nums = [3, 4, 7, 2, -3, 1, 4, 2]
k = 7
# 和が 7 になる連続部分配列:
# [3, 4]
# [7]
# [3, 4, 7, 2, -3]   (3+4+7+2-3 = 13? → NG)  ※こういうのをちゃんと排除できるかもポイント
# [7, 2, -3, 1]
# [2, -3, 1, 4, 3]? など、実際にはいくつか存在
# （出力はコードで確認）
```

## 実行結果

```
例1の入力: [1, 1, 1] k = 2
例1の結果: 2
例2の入力: [1, 2, 3] k = 3
例2の結果: 2
例3の入力: [3, 4, 7, 2, -3, 1, 4, 2] k = 7
例3の結果: 4
```
