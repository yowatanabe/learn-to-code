## 問題文

### N分以内に収まる最小本数のログファイル分割

あなたは大きなログファイルを複数の小さなファイルに分割して処理しようとしています。

整数配列 `files` が与えられ、`files[i]` は `i` 番目のログファイルのサイズを表します。

また、1回の処理では**連続したファイル群**をまとめて1本として扱う必要があります。つまり、配列をいくつかの連続部分配列に分割します。

整数 `maxSize` が与えられたとき、**各部分配列の合計サイズが `maxSize` 以下になるように分割するときに必要な最小の分割数**を返してください。

ここで、分割数とは「最終的な部分配列の本数」を意味します。

### 制約

* `1 <= len(files) <= 10^5`
* `1 <= files[i] <= 10^9`
* `1 <= maxSize <= 10^14`
* 必ず各 `files[i] <= maxSize` であるとする

## 実行例

### 例1

```python
files = [4, 2, 3, 5, 1]
maxSize = 6
```

例えば次のように分割できます。

```python
[4, 2], [3], [5, 1]
```

各部分配列の合計はそれぞれ `6`, `3`, `6` で、すべて `maxSize` 以下です。

2本に分けることはできないため、出力は

```python
3
```

### 例2

```python
files = [1, 2, 3, 4]
maxSize = 10
```

配列全体の合計が `10` なので、分割せずに1本で処理できます。

出力は

```python
1
```

### 例3

```python
files = [5, 5, 5, 5]
maxSize = 5
```

各ファイルを単独で処理するしかありません。

出力は

```python
4
```

## 実行結果

```
3
1
4
```
