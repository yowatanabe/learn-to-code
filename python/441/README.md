## 問題文

### キャッシュ容量内で処理できる最大ジョブ数

あなたはバッチ処理基盤を運用しています。整数配列 `jobs` が与えられ、`jobs[i]` は `i` 番目のジョブが使用するメモリ量を表します。

整数 `capacity` が与えられたとき、**合計メモリ使用量が `capacity` 以下になる連続したジョブ列のうち、長さが最大のもの** を求めてください。

その最大長を返してください。

### 制約

* `1 <= len(jobs) <= 10^5`
* `1 <= jobs[i] <= 10^4`
* `1 <= capacity <= 10^9`

## 実行例

### 例1

```python
jobs = [3, 1, 2, 1, 4]
capacity = 5
```

`[1, 2, 1]` の合計は `4` で、長さは `3` です。
これが最大なので、出力は

```python
3
```

### 例2

```python
jobs = [2, 2, 2, 2]
capacity = 4
```

最大で2つ連続して処理できます。

出力は

```python
2
```

### 例3

```python
jobs = [6, 7, 8]
capacity = 5
```

どのジョブも単体で `capacity` を超えるため、出力は

```python
0
```

## 実行結果

```
3
2
0
```
