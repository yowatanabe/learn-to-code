# No.299

## 問題: 暗号メッセージのデコード通り数（Decode Ways）

数字のみからなる文字列 `s` が与えられます。

この文字列は次のルールで英大文字にデコードされた暗号メッセージを表しています。

* `'1'` → `'A'`
* `'2'` → `'B'`
* …
* `'26'` → `'Z'`

あなたのタスクは、

> 文字列 `s` を **有効な方法でデコードできる通り数** を求めてください。

ただし:

* `'0'` 単体はどの文字にも対応しないので **無効** です。
* ただし `'10'`, `'20'` のように `'1'` または `'2'` と組み合わさっている場合は有効です。
* 通り数が大きくなる可能性がありますが、ここではそのまま整数として返してよいものとします。

### 制約（目安）

* `1 ≤ len(s) ≤ 100`
* `s` は `'0'`〜`'9'` のみからなる

## 実行例

```python
# 例1
s = "12"
# "12" のデコード方法:
# "1" "2" -> "AB"
# "12"    -> "L"
# 合計 2 通り
# 出力: 2

# 例2
s = "226"
# デコード方法:
# "2" "2" "6" -> "B" "B" "F"
# "22" "6"    -> "V" "F"
# "2" "26"    -> "B" "Z"
# 合計 3 通り
# 出力: 3

# 例3
s = "06"
# "0" からは始められない → そもそもデコード不可能
# 出力: 0

# 例4
s = "11106"
# 有効なデコードいくつか:
# "11" "10" "6"  -> K J F
# "1"  "10" "6"  -> A J F
# など合計 2 通り
# 出力: 2
```

## 実行結果

```
例1の入力: 12
例1の結果: 2
例2の入力: 226
例2の結果: 3
例3の入力: 06
例3の結果: 0
例4の入力: 11106
例4の結果: 2
```
