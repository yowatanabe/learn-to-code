## 問題文（LeetCode風・日本語）

### 問題：アラートの重複抑制（クールダウン付き最小実行時間）

同じ種類のアラートを短時間に連続送信するとノイズになるため、
**同じ種類のアラートは、送信後少なくとも `n` 秒空けないと再送できない**（クールダウン）というルールがあるとします。

送信したいアラートの種類が配列 `tasks`（各要素は大文字英字 `A-Z`）で与えられます。

1秒に送信できるアラートは高々1件です。送信できない秒は `idle`（待機）として空けられます。

すべてのアラートを送信し終わるまでに必要な**最小の経過秒数**を返してください。

* `1 <= len(tasks) <= 10^5`
* `0 <= n <= 10^5`

**求めるもの**：最小経過秒数（整数）

## 実行例

例1:

* 入力: `tasks = ["A","A","A","B","B","B"]`, `n = 2`
* 説明: `A B idle A B idle A B` → 8秒
* 出力: `8`

例2:

* 入力: `tasks = ["A","A","A","B","B","B"]`, `n = 0`
* 説明: クールダウンなしなので連続で送れる → 6秒
* 出力: `6`

例3:

* 入力: `tasks = ["A","A","A","A","B","C","D"]`, `n = 2`
* 説明: 最小は 10秒
* 出力: `10`

## 実行結果

```
8
6
10
```
