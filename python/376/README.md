# No.376

## 問題文（LeetCode風・日本語）

**問題：ログの相関IDを統合（最小置換回数）**

あなたは分散システムのログを解析しています。

ログには相関ID（correlation id）が文字列として記録されていますが、サービスごとに表記揺れがあり、同一視したいIDのグループがあります。

長さ `n` の配列 `ids` が与えられます。さらに、`pairs` という配列が与えられ、`pairs[i] = [a, b]` は **ID `a` と `b` は同一グループとして扱う（相互に変換可能）**ことを意味します。

この関係は推移的です（例：`a~b` かつ `b~c` なら `a~c`）。

あなたは `ids` の各要素を「同一グループ内の任意のID」に置き換えることができます。

最終的に `ids` の全要素を **完全に同じ文字列** にしたいとき、必要な**最小置換回数**を求めてください。

不可能な場合は `-1` を返してください。

* 置換は「要素を別IDに変更したら1回」と数える（同じIDのままなら0回）
* `pairs` に出てこないIDはそれ単体のグループ
* `1 <= n <= 2 * 10^5`
* `0 <= len(pairs) <= 2 * 10^5`
* 文字列は英数字/記号を含むが比較は完全一致

**求めるもの**：最小置換回数（整数）、不可能なら `-1`

## 実行例

例1:

* 入力:

  * `ids = ["a1", "b1", "c1", "a1"]`
  * `pairs = [["a1","b1"], ["b1","c1"]]`
* 説明:

  * `a1, b1, c1` は同一グループ
  * 全部 `a1` に統一できる。すでに `a1` は2つあるので、残り2つを置換 → 2回
* 出力: `2`

例2:

* 入力:

  * `ids = ["x", "y", "y"]`
  * `pairs = []`
* 説明:

  * `x` と `y` は別グループなので全て同じにできない
* 出力: `-1`

例3:

* 入力:

  * `ids = ["svc-A", "svc_B", "svc-A", "svc_B"]`
  * `pairs = [["svc-A","svc_B"]]`
* 説明:

  * 同一グループ。`svc-A` か `svc_B` に統一できる
  * すでに各2個あるので、最小置換は2回
* 出力: `2`

## 実行結果

```
2
-1
2
```
