# No.342

## 問題: 監視アラートの「連続発火」を圧縮せよ（String Compression with Counts）

監視アラートのログは同じ種類が連続することがあります。

文字列 `s` が与えられます。`s` は英小文字のみからなります。

> 文字列を **ランレングス圧縮（Run-Length Encoding）** してください。
> 連続する同じ文字を `文字 + 回数` で表し、圧縮後の文字列を返してください。
> ただし **回数が 1 の場合は数字を付けない**（例: `a` は `a` のまま）。

例:

`"aaabccdddd"` → `"a3bc2d4"`

### 制約（目安）

* `1 ≤ len(s) ≤ 10^5`
* `s` は `a`〜`z`

時間計算量 **O(n)** を目指してください。

## 実行例

```python
# 例1
s = "aaabccdddd"
# aが3回, bが1回, cが2回, dが4回
# 出力: "a3bc2d4"

# 例2
s = "abcd"
# すべて1回なので数字は付かない
# 出力: "abcd"

# 例3
s = "zzzzzz"
# 出力: "z6"
```

## 実行結果

```
例1 入力: s='aaabccdddd'
例1 出力: 'a3bc2d4' (期待値: 'a3bc2d4')
--------------------------------------------------
例2 入力: s='abcd'
例2 出力: 'abcd' (期待値: 'abcd')
--------------------------------------------------
例3 入力: s='zzzzzz'
例3 出力: 'z6' (期待値: 'z6')
--------------------------------------------------
```
