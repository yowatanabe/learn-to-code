# No.377

## 問題文（LeetCode風・日本語）

**問題：デプロイ順序の最短時間（依存関係付き・並列実行）**

あなたは複数のマイクロサービスをデプロイします。

サービス `i` のデプロイには `time[i]` 分かかります。

また、依存関係が `deps` として与えられます。`deps[j] = [a, b]` は **サービス `a` のデプロイが完了してからでないと、サービス `b` のデプロイを開始できない**ことを意味します（有向辺 `a -> b`）。

あなたは**同時にいくつでも**デプロイを並列実行できます（リソースは十分あるものとする）。

すべてのサービスをデプロイ完了するのに必要な**最短時間**を求めてください。

もし依存関係にサイクルがあり、全てを完了できない場合は `-1` を返してください。

* `1 <= n <= 2 * 10^5`
* `len(deps) <= 2 * 10^5`
* `1 <= time[i] <= 10^9`
* サービスは `0..n-1`

**求めるもの**：最短完了時間（整数）、不可能なら `-1`

## 実行例

例1:

* 入力:

  * `n = 4`
  * `time = [3, 2, 5, 4]`
  * `deps = [[0, 2], [1, 2], [2, 3]]`
* 説明:

  * 0と1は並列に開始できる（3分と2分）
  * 2は0と1の両方完了後に開始 → max(3,2)=3 分後に開始、5分かかるので完了は8分
  * 3は2の後 → 8分後に開始、4分で完了 → 12分
* 出力: `12`

例2:

* 入力:

  * `n = 3`
  * `time = [2, 2, 2]`
  * `deps = [[0, 1], [1, 2], [2, 0]]`
* 説明: サイクルがあるので不可能
* 出力: `-1`

## 実行結果

```
12
-1
```
