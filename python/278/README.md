# No.278

## 問題文

**最小の値を持つペア**

整数の配列 `nums` と整数 `k` が与えられたとき、配列 `nums` の中で **和が `k` となる異なる整数のペア** の個数を求めなさい。

* 例えば、`nums[i] + nums[j] = k` となる `(i, j)` の組み合わせを探します（`i != j`）。
* 同じペアを複数回カウントしないようにしてください。
* `nums` に含まれる整数の順序は関係なく、ペアの順序も無視します（`(1, 2)` と `(2, 1)` は同じペアとして数える）。
* 配列の長さ `n` は `1 ≤ n ≤ 2 * 10^5` で、要素は `-10^9 ≤ nums[i] ≤ 10^9` です。

## 実行例

**例1**

```text
入力:
nums = [1, 2, 3, 4, 5], k = 5

出力:
2

説明:
ペアは (1, 4) と (2, 3) の2つです。
```

**例2**

```text
入力:
nums = [1, 2, 2, 3, 4], k = 4

出力:
2

説明:
ペアは (1, 3) と (2, 2) の2つです。
```

**例3**

```text
入力:
nums = [1, 1, 1, 1], k = 2

出力:
1

説明:
ペア (1, 1) は一度しかカウントされません。
```

**例4**

```text
入力:
nums = [1, 2, 3], k = 6

出力:
0

説明:
和が6になるペアは存在しません。
```

## 実行結果

```
2
2
1
0
```
