## 問題: 並列実行できる最大数（Maximum Concurrent Tasks）

SREのジョブ実行（例: バッチ、デプロイ、メンテ）を想定します。

各タスクは実行区間 `[start, end)`（**start は含む、end は含まない**）で表されます。

* 例: `[1,5)` と `[5,7)` は **重ならない**（同時実行ではない）

整数のペア配列 `intervals` が与えられたとき、

> 任意の時刻に同時実行されているタスク数の最大値（最大同時実行数）を返してください。

### 制約（目安）

* `1 ≤ len(intervals) ≤ 10^5`
* `0 ≤ start < end ≤ 10^9`

時間計算量 **O(n log n)** を目指してください。

## 実行例

```python
# 例1
intervals = [[1,5],[2,6],[4,8],[8,9]]
# 時刻4付近で [1,5), [2,6), [4,8) が重なり最大3
# 出力: 3

# 例2
intervals = [[0,10],[10,20],[20,30]]
# 端でつながるだけで重ならない
# 出力: 1

# 例3
intervals = [[3,7],[1,4],[2,5]]
# 時刻3付近で3本重なる
# 出力: 3
```

## 実行結果

```

例1 出力: 3 (期待値: 3)

例2 出力: 1 (期待値: 1)
------------------------------------------------------------
例3 入力: intervals=[[3, 7], [1, 4], [2, 5]]
例3 出力: 3 (期待値: 3)
------------------------------------------------------------
```
