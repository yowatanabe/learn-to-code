## 問題文（LeetCode風・日本語）

**問題：リトライ待ち時間の最小化（最小合計待ち）**

あなたはSREとして、複数のジョブに対して「リトライ待ち時間（秒）」を設定する必要があります。

各ジョブ `i` には、待ち時間として使える候補値が昇順の配列 `options[i]` として与えられます。

ただし、**同じ待ち時間（秒）を複数のジョブで使うことは禁止**とします（衝突するとスロットリングや一斉リトライが起きるため）。

各ジョブから待ち時間を1つずつ選び、すべての選択値が相異なるようにしたときの、**合計待ち時間の最小値**を求めてください。

不可能な場合は `-1` を返してください。

* `1 <= n <= 2 * 10^5`（ジョブ数）
* `sum(len(options[i])) <= 2 * 10^5`
* `options[i]` は昇順、要素は非負整数

**求めるもの**：最小合計待ち時間（整数）、不可能なら `-1`

## 実行例

例1:

* 入力:

  * `options = [[1, 2, 4], [1, 3], [2, 3, 5]]`
* 選び方の一例: ジョブ0→1, ジョブ1→3, ジョブ2→2（全て異なる）
* 合計: 1 + 3 + 2 = 6（これが最小）
* 出力: `6`

例2:

* 入力:

  * `options = [[0], [0]]`
* 同じ値0を2回使う必要があり不可能
* 出力: `-1`

例3:

* 入力:

  * `options = [[2, 2, 2], [2], [2, 3]]`
* ジョブ0→2, ジョブ2→3 は可能だが、ジョブ1は2しかなく衝突する
* 出力: `-1`

## 実行結果

```
6
-1
-1
```
