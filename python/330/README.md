# No.330

## 問題

あなたはサービスのアクセスログを解析している。各ログは `[start, end]`（いずれも**秒**、**両端含む**）で表され、ユーザーがその区間ずっと「接続中」とみなされる。

同時接続数（アクティブな区間数）が最大になる時刻を求めたい。

最大同時接続数 `max_count` と、その `max_count` を達成する**最も早い時刻** `t` を返せ。

* `intervals[i] = [start_i, end_i]`
* `start_i <= end_i`
* 端点は**含む**（例：`[1,1]` は時刻1に1ユーザー接続）
* 出力は `(t, max_count)` の形式でよい

**制約**

* `1 <= len(intervals) <= 2 * 10^5`
* `0 <= start_i, end_i <= 10^9`

## 実行例

### 例1

入力: `[[1,4],[2,4],[4,6],[5,5]]`
出力: `(4, 3)`
説明: 時刻4は `[1,4]`,`[2,4]`,`[4,6]` が同時にアクティブ。

### 例2

入力: `[[10,10],[10,12],[13,15]]`
出力: `(10, 2)`

### 例3

入力: `[[0,2],[1,3],[2,4]]`
出力: `(2, 3)`
説明: 両端含むため、時刻2は3つすべてがアクティブ。

## 実行結果

```
intervals=[[1, 4], [2, 4], [4, 6], [5, 5]] -> (4, 3) (expected (4, 3))
intervals=[[10, 10], [10, 12], [13, 15]] -> (10, 2) (expected (10, 2))
intervals=[[0, 2], [1, 3], [2, 4]] -> (2, 3) (expected (2, 3))
```
