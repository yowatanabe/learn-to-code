# No.380

## 問題文（LeetCode風・日本語）

**問題：ログ集計ウィンドウの最小サイズ（目標件数を満たす最短区間）**

あなたはSREとして、時系列ログの「あるイベント」の発生回数を監視しています。

1秒ごとの発生回数を表す配列 `cnt` が与えられます（`cnt[i]` は i 秒目の発生回数）。

運用上、連続した時間帯（区間）を1つ選び、その区間内の合計発生回数が `target` 以上になるようにしたいです。

そのような区間のうち、**長さ（秒数）が最小**のものを求めてください。存在しない場合は `0` を返してください。

* `1 <= len(cnt) <= 2 * 10^5`
* `0 <= cnt[i] <= 10^9`
* `1 <= target <= 10^18`

**求めるもの**：条件を満たす最小の区間長（整数）。存在しなければ `0`。

## 実行例

例1:

* 入力: `cnt = [2, 3, 1, 2, 4, 3]`, `target = 7`
* 説明: 区間 `[4, 3]` の合計は7で長さ2が最小
* 出力: `2`

例2:

* 入力: `cnt = [1, 1, 1, 1]`, `target = 10`
* 説明: どの区間でも合計が10に届かない
* 出力: `0`

例3:

* 入力: `cnt = [7]`, `target = 7`
* 説明: 全体区間で達成、長さ1
* 出力: `1`

## 実行結果

```
2
0
1
```
