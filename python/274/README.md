# No.273

## 問題文

**連続部分配列の合計が `k` となる個数**

整数配列 `nums` と整数 `k` が与えられる。
**連続部分配列（空でない）** のうち、要素の合計が **ちょうど `k`** になるものの**個数**を求めよ。

* `0 ≤ len(nums) ≤ 2 * 10^5`
* `-10^9 ≤ nums[i], k ≤ 10^9`
* 期待計算量: **O(n)**、追加メモリ: **O(n)**

**ヒント**:

* 前からの累積和 `prefix` を取り、各位置で「`prefix - k` が過去に何回出たか」を数えて足し込む（ハッシュマップ）。
* 初期状態として「累積和 0 が 1 回出現した」と見なすと、先頭からの部分配列も自然に数えられる。

## 実行例

**例1**

```text
入力:
nums = [1, 1, 1]
k = 2
出力:
2

説明:
[1,1] (0..1), [1,1] (1..2) の2つ。
```

**例2**

```text
入力:
nums = [1, 2, 3]
k = 3
出力:
2

説明:
[1,2] (0..1), [3] (2..2) の2つ。
```

**例3**

```text
入力:
nums = [3, 4, 7, -2, 2, 1, 4, 2]
k = 7
出力:
6

説明:
和が7の連続部分配列は:
[3,4], [7], [7,-2,2], [-2,2,1,4,2], [2,1,4], [1,4,2] の6つ。
```

**例4**

```text
入力:
nums = []
k = 0
出力:
0
```

## 実行結果

```
2
2
6
0
```
