## 問題: 設定変更の衝突を避けて最小削除（Non-overlapping Intervals）

デプロイやメンテナンスの「実行区間」が `intervals[i] = [start, end]` で与えられます。

ある2つの区間が重なっている場合（`[a,b]` と `[c,d]` が `c < b` を満たすなど）、同時実行できず衝突します。

> 重なりが一切ないようにするために、削除すべき区間の **最小数** を返してください。

※ ここでは **[start, end)**（endは含まない）として扱い、
`[1,2)` と `[2,3)` は **重ならない** とします。

### 制約（目安）

- `1 ≤ len(intervals) ≤ 10^5`
- `-10^9 ≤ start < end ≤ 10^9`

時間計算量 **O(n log n)** を目指してください。

---

## 実行例

```python
# 例1
intervals = [[1,2],[2,3],[3,4],[1,3]]
# [1,3] を削除すれば残りは重ならない
# 出力: 1

# 例2
intervals = [[1,2],[1,2],[1,2]]
# 2つ削除して1つ残す
# 出力: 2

# 例3
intervals = [[1,2],[2,3]]
# 端で接するだけで重ならない
# 出力: 0
```

## 実行結果

```
例1 入力: intervals=[[1, 2], [2, 3], [1, 3], [3, 4]]
例1 出力: 1 (期待値: 1)
------------------------------------------------------------
例2 入力: intervals=[[1, 2], [1, 2], [1, 2]]
例2 出力: 2 (期待値: 2)
------------------------------------------------------------
例3 入力: intervals=[[1, 2], [2, 3]]
例3 出力: 0 (期待値: 0)
------------------------------------------------------------
```
