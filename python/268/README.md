# No.268

## 問題文

**最長増加トラフィック期間**

あるサービスのリクエスト数（1分ごと）が整数配列 `traffic` として与えられる。

`traffic[i]` は i 分目のリクエスト数を表す。

**インデックスが昇順の一部を抜き出して作れる部分列のうち、値が「厳密に増加する（`<`）」ものの最長長さ**を求めよ。

要するに、`i1 < i2 < ... < ik` で

`traffic[i1] < traffic[i2] < ... < traffic[ik]` となるような k の最大値を求める。

* `0 ≤ len(traffic) ≤ 2 * 10^5`
* `-10^9 ≤ traffic[i] ≤ 10^9`
* 期待計算量: **O(n log n)**、追加メモリ: **O(n)**

**注意:**

* 「部分列」であり「部分配列」ではないので、要素は**飛び飛び**に選んでよい（連続でなくてよい）。
* `<=` ではなく **`<`（厳密な増加）** である点に注意。

## 実行例

**例1**

```text
入力:
traffic = [10, 9, 2, 5, 3, 7, 101, 18]

出力:
4

説明:
最長の増加部分列の一つは [2, 3, 7, 18]（長さ4）。
他にも [2,5,7,101] などがある。
```

**例2**

```text
入力:
traffic = [0, 1, 0, 3, 2, 3]

出力:
4

説明:
最長の増加部分列の一つは [0,1,2,3]（長さ4）。
```

**例3**

```text
入力:
traffic = [7, 7, 7, 7]

出力:
1

説明:
厳密な増加なので、同じ値を2つ以上は取れない。任意の1要素だけ選んで長さ1。
```

**例4**

```text
入力:
traffic = []

出力:
0
```

## 実行結果

```
4
4
1
0
```
