# No.265

## 問題文

**K個まで障害を通過できる最短経路**

2次元グリッド `grid` が与えられる。

`grid[r][c]` は以下のいずれか:

* `0` : 通過可能なセル
* `1` : 障害セル（通常は通れないが「踏み抜き」可能）

あなたは左上 `(0, 0)` から右下 `(m-1, n-1)` まで移動したい。

1ステップで、**上下左右に隣接するセル**のいずれかに移動できる。

ただし、最大 `K` 個までなら障害セル (`1`) を「踏み抜いて」通過してよい（通過した時点でカウントされる）。

`K` を超えて障害セルを通ることはできない。

このとき、出発セルをステップ数 0 として、目的地に到達するまでの**最短ステップ数**を求めよ。

到達できない場合は `-1` を返せ。

* `grid` のサイズは `m × n` (`1 ≤ m, n ≤ 40` 程度)
* 各 `grid[r][c]` は `0` または `1`
* `0 ≤ K ≤ m * n`
* 出発セル `(0,0)` および目的地 `(m-1,n-1)` が障害セル (`1`) の場合もある

  * その場合も、踏み抜きカウントに含めて進める（ただし `K` が足りないと即座に不可能）

**要求される計算量:**

* 時間計算量: おおよそ `O(m * n * K)`
* 追加メモリ: 同程度

## 実行例

**例1**

```
入力:
grid =
[
  [0, 0, 0],
  [1, 1, 0],
  [0, 0, 0]
]
K = 1

出力:
4

説明:
最短経路の一つ:
(0,0) -> (0,1) -> (0,2) -> (1,2) -> (2,2)
長さ4。障害は踏まなくてもよいが、他ルートでは1個まで踏める。
```

**例2**

```
入力:
grid =
[
  [0, 1, 0],
  [1, 1, 0],
  [0, 0, 0]
]
K = 2

出力:
4

説明:
例えば以下の経路で障害を2つ踏み抜いて到達できる:
(0,0) -> (0,1) # ここで1個使用
        -> (1,1) # ここで2個使用
        -> (1,2) -> (2,2)
ステップ数は4。
```

**例3**

```
入力:
grid =
[
  [0, 1, 1],
  [1, 1, 1],
  [1, 1, 0]
]
K = 1

出力:
-1

説明:
障害だらけで、1個しか踏み抜けないため到達不能。
```

**例4**

```
入力:
grid =
[
  [1, 0],
  [0, 0]
]
K = 1

出力:
2

説明:
開始セル (0,0) が障害なので、そこで1個消費。
その後 (1,0) -> (1,1) の2ステップでゴール。
ステップ数は2。
```

## 実行結果

```
4
4
-1
2
```
