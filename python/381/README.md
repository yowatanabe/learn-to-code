# No.381

## 問題文（LeetCode風・日本語）

**問題：メンテナンス時間の選定（最小の最大負荷）**

あなたはSREとして、メンテナンス作業を `m` 日間に分割して実施します。

連続するリクエスト量の配列 `load` が与えられ、`load[i]` は i 番目の時間スロットの負荷（整数）です。

あなたは `load` を **連続した区間に分割**して、ちょうど `m` 個のグループにします（各要素は必ずどれか1つのグループに属し、グループは連続区間）。

各グループの負荷は、そのグループ内の合計値です。

「最も重いグループの負荷（最大グループ和）」をできるだけ小さくしたいです。

`load` をちょうど `m` 個に分割したときの **最大グループ和の最小値**を返してください。

* `1 <= len(load) <= 2 * 10^5`
* `1 <= m <= len(load)`
* `1 <= load[i] <= 10^9`

**求めるもの**：最大グループ和の最小値（整数）

## 実行例

例1:

* 入力: `load = [7, 2, 5, 10, 8]`, `m = 2`
* 説明: `[7,2,5]` と `[10,8]` に分けると最大は `max(14,18)=18`。これが最小。
* 出力: `18`

例2:

* 入力: `load = [1, 2, 3, 4, 5]`, `m = 2`
* 説明: `[1,2,3]` と `[4,5]` → 最大 `max(6,9)=9`
* 出力: `9`

例3:

* 入力: `load = [1, 4, 4]`, `m = 3`
* 説明: `[1] [4] [4]` しかないので最大は4
* 出力: `4`

## 実行結果

```
18
9
4
```
