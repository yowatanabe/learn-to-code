# No.403

## 問題：設定値を使い切る最小ラウンド数

あなたは SRE として、複数の設定変更タスクを処理しています。

各タスクには種類を表す整数IDがあり、同じIDのタスクは同じ種類です。

1 回のラウンドで処理できるのは、**同じ種類のタスク 2 個** または **3 個** です。

異なる種類のタスクを同じラウンドでまとめて処理することはできません。

整数配列 `tasks` が与えられたとき、すべてのタスクを処理するために必要な**最小ラウンド数**を求めてください。

もし不可能な場合は `-1` を返してください。

* `1 <= len(tasks) <= 2 * 10^5`
* `1 <= tasks[i] <= 10^9`

**求めるもの**：最小ラウンド数（整数）、不可能なら `-1`

## 実行例

例1:

* 入力: `tasks = [2,2,3,3,2,4,4,4,4,4]`
* 説明:

  * `2` は 3 個あるので 1 ラウンド（3個）
  * `3` は 2 個あるので 1 ラウンド（2個）
  * `4` は 5 個あるので 2 ラウンド（3個 + 2個）
  * 合計 4 ラウンド
* 出力: `4`

例2:

* 入力: `tasks = [1,1,2,2,2,3]`
* 説明:

  * `1` は 2 個 → 1 ラウンド
  * `2` は 3 個 → 1 ラウンド
  * `3` は 1 個 → 2個または3個でしか処理できないので不可能
* 出力: `-1`

例3:

* 入力: `tasks = [5,5,5,5]`
* 説明:

  * `5` は 4 個あるので 2 ラウンド（2個 + 2個）
* 出力: `2`

## 実行結果

```
4
-1
2
```
