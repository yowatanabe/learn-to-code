## 問題: 依存関係のあるタスクの実行順（Course Schedule II 風）

`0` から `n-1` の `n` 個のタスクがあります。

依存関係 `prerequisites` が与えられ、`prerequisites[i] = [a, b]` はタスク `a` を実行するには、先にタスク `b` を完了している必要があることを表します。

**すべてのタスクを完了できるなら**、可能な実行順序のうち1つを配列で返してください。
**不可能（依存関係にサイクルがある）なら**空配列 `[]` を返してください。

### 制約（目安）

* `1 ≤ n ≤ 10^5`
* `0 ≤ len(prerequisites) ≤ 2 * 10^5`
* `0 ≤ a, b < n`
* 同じペアが重複して与えられることもある（重複は無視して良い）

期待解法: トポロジカルソート（BFS/Kahn 法） `O(n + m)`

## 実行例

```python
# 例1
n = 4
prerequisites = [[1,0],[2,0],[3,1],[3,2]]
# 0 -> 1, 0 -> 2, 1 -> 3, 2 -> 3
# 可能な順序: [0,1,2,3] や [0,2,1,3]
# 出力例: [0,2,1,3]

# 例2
n = 2
prerequisites = [[1,0],[0,1]]
# 0 と 1 が互いに依存していてサイクル
# 出力: []

# 例3
n = 3
prerequisites = []
# 依存なしなので任意順
# 出力例: [0,1,2]
```

## 実行結果

```
例1 入力: n=4, prerequisites=[[1, 0], [2, 0], [3, 1], [3, 2]]
例1 出力: [0, 1, 2, 3]
--------------------------------------------------
例2 入力: n=2, prerequisites=[[1, 0], [0, 1]]
例2 出力: []
--------------------------------------------------
例3 入力: n=3, prerequisites=[]
例3 出力: [0, 1, 2]
--------------------------------------------------
```
